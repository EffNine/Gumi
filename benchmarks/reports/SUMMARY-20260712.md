## Executive Summary

Gumi improves local 9B model reliability by **100% for JSON output** and **22-33% for instruction following** with minimal latency overhead (+60ms p50).

Sprint 12 fixes eliminated the dominant failure modes: **0 HTTP errors** (was ~50% due to LM Studio model-loading 502s), **0 repetition false positives** (was 113 validation failures), and validation failures are now fully diagnosable via the telemetry database. A post-fix JSON repair improvement now handles any language-tagged code fence (```python, ```javascript, etc.), bringing all three tested models to **100% JSON validity in stabilized and structured modes**.

When combined with an agent framework, Gumi's per-turn reliability improvements compound across multi-turn agentic coding sessions — this is the mechanism by which Gumi helps local models approach the benchmark scores achieved by cloud agent frameworks.

## 6. Sprint 12: Infrastructure Fixes

### Retry Backoff (P0)

The `generateOnce` function now uses exponential backoff:
- **Model-loading errors** (LM Studio `"Engine protocol startup was aborted"`): 3s → 6s
- **Other retryable errors**: 2s → 4s
- `response_format` is only stripped on retry when the error mentions format rejection

### Validation & Repair Telemetry (P1)

Two new telemetry methods write to the existing `validation_reports` and `repair_reports` tables (previously empty). Validation failures now include issue details in `errors.details_json`:

```json
{"cause":"REPETITION at choices[0].message.content: assistant response contains repeated lines or sentences"}
```

### JSON-Aware Repetition Detection (P2)

`hasRepetition` skips detection for JSON output (`json_object`, `json_schema`, `structured` mode, valid JSON content). Plain-text repetition detection unchanged.

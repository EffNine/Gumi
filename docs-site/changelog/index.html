v0.2.0-alpha · 2026-07-14

First tagged release: agent mode, memory, model management

This is the first public alpha. It bundles 13 sprints of runtime work into something you can drop between your coding agent and a local model.

New

  • Agent mode — step budgets, tool-call loop detection, JSON validation, and sliding-window context compaction.
  • Agentic coding router — per-step model routing by difficulty and task type, no extra inference.
  • Memory engine — SQLite facts, episode summaries, and model-fit tracking shared across all models.
  • LM Studio model management — load, unload, list models via LM Studio's v1 REST API.
  • Instruction-following assist — detects 14 constraint types and enforces them post-generation.
  • Managed thinking — reasoning budgets, stripping, and auto-disable for JSON/tool workflows.

Reliability fixes (P0–P3)

  • P0: Exponential backoff for LM Studio 502s — ~50% HTTP errors → 0%.
  • P1: Validation telemetry now records issue code, message, and location.
  • P2: Repetition detection skips JSON/structured output — 113 false positives → 0.
  • P3: JSON repair handles any language-tagged code fence.

Roadmap

What's next

The runtime is solid for an alpha. The next phase is about smoothing adoption and hardening edge cases.

Near term (v0.3.0-alpha)

  • Provider health checks and automatic failover in the dashboard.
  • Structured-output mode hardening (schema enforcement in retry prompts).
  • Profile Doctor goes from read-only to auto-fix suggestions.
  • Continue.dev tab-autocomplete path tested and documented.

Medium term (v0.4.0-alpha and beyond)

  • More provider adapters beyond LM Studio and Ollama.
  • Streaming parity across all pipeline modes.
  • Memory engine search and fact editing UI.
  • Public model-profile registry so users don't have to hand-tune YAML.

  • Sprints 0–9 — Gateway, telemetry, dashboard, prompt repair, validation, and tool shims. See CHANGELOG.md for the full history.

Quickstart

You'll have Gumi running in about two minutes. You just need a local inference engine (LM Studio or Ollama) with a model loaded.

1. Build and start

Terminal
git clone https://github.com/EffNine/Gumi.git
cd Gumi
make build
./gumi start

Gumi starts on http://127.0.0.1:8787 (API) and http://127.0.0.1:8788 (dashboard). Leave this terminal open.

2. Open the dashboard

Navigate to http://127.0.0.1:8788. The dashboard has 11 pages covering every aspect of the runtime:

  • Overview — Runtime status, pipeline visualization, provider health
  • Playground — Interactive chat with provider/model/mode selection
  • Requests — Request history with filtering and status indicators
  • Analytics — Latency distribution, provider breakdown, success rate
  • Providers — Provider health cards with model counts
  • Models — Model listing with load/unload and configuration
  • Memory — Facts CRUD, model-fit leaderboard, memory engine status
  • Profiles — Model profile listing with capabilities
  • Logs — Real-time log viewer via SSE with level filtering
  • Config — Resolved config viewer with redacted secrets
  • Doctor — Visual diagnostic checks with suggestions

3. Call the API

Any OpenAI-compatible client works out of the box:

cURL
curl http://127.0.0.1:8787/v1/chat/completions \
  -H "Authorization: Bearer gumi-local" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "lmstudio:qwen2.5-coder-7b-instruct",
    "messages": [{"role": "user", "content": "Write a Go function that adds two ints. Code only."}]
  }'
Python
from openai import OpenAI

client = OpenAI(base_url="http://127.0.0.1:8787/v1", api_key="gumi-local")
response = client.chat.completions.create(
    model="lmstudio:qwen2.5-coder-7b-instruct",
    messages=[{"role": "user", "content": "Write a tiny TypeScript add function."}],
)
print(response.choices[0].message.content)

4. Model naming

When routing through Gumi, prefix the model name with the provider:

  • lmstudio:qwen2.5-coder-7b-instruct — LM Studio
  • ollama:llama3.2:3b — Ollama
  • local:auto — Auto-select the first available provider

5. Pipeline modes

Gumi supports five runtime modes. Pass them via the gumi extension in your request:

Request with mode selection
{
  "model": "ollama:llama3.2:3b",
  "messages": [{"role": "user", "content": "What is 2+2?"}],
  "gumi": {
    "mode": "stabilized"
  }
}
ModeDescription
directThin proxy — no processing
lightweightProfile defaults + anti-loop guard only
stabilizedFull pipeline: context, prompt, validation, repair, telemetry
structuredStabilized + strict JSON/schema output enforcement
agentStabilized + step budget, tool-call loop detection, context compaction

6. Configuration

Copy the example config and customize:

Terminal
cp gumi.example.yaml ~/.gumi/gumi.yaml
# Edit ~/.gumi/gumi.yaml to set your default provider and model

Environment variables override YAML values:

  • GUMI_PROVIDER_DEFAULT=lmstudio
  • GUMI_LMSTUDIO_URL=http://192.168.0.164:1234/v1
  • GUMI_DEFAULT_MODEL=ornith-1.0-9b@q4_k_m
  • GUMI_PROVIDER_TIMEOUT_SECONDS=120

What it does

🔧

Fixes broken JSON

Local models love to produce almost-JSON. Gumi catches it, fixes it, and retries — so your agent loops don't grind to a halt over a missing comma.

🎯

Helps models follow instructions

Small models forget what you asked for. Gumi spots 14 kinds of constraints in your prompt and gently reminds the model before it answers.

🔄

Stops repetition loops

Models get stuck in loops. Gumi detects repeated output, repeated tool calls, and context overflow — and breaks the cycle before you do.

🧠

Remembers what matters

Zero-VRAM memory that survives model swaps and session restarts. Facts, episode summaries, and model-fit data in SQLite — no GPU needed.

🚀

Picks the right model per step

A "fix typo" step routes to a tiny fast model. "Implement payment handler" escalates to a strong one. All automatic, all per-agent-step.

📊

Shows you what's happening

11-page dashboard with live logs, request analytics, provider health, and diagnostics. Everything stays on your machine.

Who is this for?

You run a coding agent locally

OpenCode, Continue, Cline — you've tried them with a local model and hit broken JSON, repeated output, or empty responses. Point your client at Gumi instead of the raw provider and those problems disappear.

You use Ollama or LM Studio

You like running models locally but they feel rough in real apps. Gumi adds JSON repair, instruction help, anti-loop guards, and a dashboard — without replacing your model or your workflow.

You're building something on local AI

You need an OpenAI-compatible reliability layer with model routing, memory, and provider-specific fixes. Gumi is that layer — one less thing for you to build.

Quick start

Terminal
git clone https://github.com/EffNine/Gumi.git
cd Gumi
make build
./gumi start
cURL
curl http://127.0.0.1:8787/v1/chat/completions \
  -H "Authorization: Bearer gumi-local" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "lmstudio:qwen2.5-coder-7b-instruct",
    "messages": [{"role": "user", "content": "Write a Go function that adds two ints. Code only."}]
  }'
Python
from openai import OpenAI

client = OpenAI(base_url="http://127.0.0.1:8787/v1", api_key="gumi-local")
response = client.chat.completions.create(
    model="lmstudio:qwen2.5-coder-7b-instruct",
    messages=[{"role": "user", "content": "Write a tiny TypeScript add function."}],
)
print(response.choices[0].message.content)

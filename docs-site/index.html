What makes Gumi different

🔧

JSON validation + repair

Local models often produce broken JSON. Gumi detects, repairs, and retries automatically — no more parsing failures in your agent loops.

🎯

Instruction-following assist

Auto-detects 14 constraint types from user prompts and injects explicit reminders. Post-generation validation with automatic retry.

🔄

Anti-loop detection

Detects repeated output, tool-call loops, and context overflow. Injects loop-break hints before the model spirals.

🧠

Memory engine

Zero-VRAM persistent memory for coding agents. Facts, episode summaries, and model-fit data stored in SQLite — shared across all models.

🚀

Agentic coding router

Automatic per-step model selection by difficulty and task type. A "fix typo" routes to a tiny model; "implement payment handler" escalates to a strong one.

📊

Local telemetry dashboard

11-page dashboard with real-time logs, request analytics, provider health, model management, and diagnostic checks — all local, all private.

Who is this for?

OpenCode / Continue / Cline users

You run a coding agent on a local model and hit broken JSON, repeated output, or empty responses. Point your client at Gumi instead of the raw provider and those failure modes disappear.

Ollama / LM Studio users

You like local inference but the model is rough in real apps. Gumi adds JSON repair, instruction-following assist, anti-loop guards, and telemetry without replacing your model.

Local AI app builders

You're building on top of local models and need an OpenAI-compatible reliability layer with model routing, memory, and provider-specific fixes. Gumi is that layer.

Quick start

Terminal
git clone https://github.com/EffNine/Gumi.git
cd Gumi
make build
./gumi start
cURL
curl http://127.0.0.1:8787/v1/chat/completions \
  -H "Authorization: Bearer gumi-local" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "lmstudio:qwen2.5-coder-7b-instruct",
    "messages": [{"role": "user", "content": "Write a Go function that adds two ints. Code only."}]
  }'
Python
from openai import OpenAI

client = OpenAI(base_url="http://127.0.0.1:8787/v1", api_key="gumi-local")
response = client.chat.completions.create(
    model="lmstudio:qwen2.5-coder-7b-instruct",
    messages=[{"role": "user", "content": "Write a tiny TypeScript add function."}],
)
print(response.choices[0].message.content)
